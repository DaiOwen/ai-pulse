# AI 脉搏 · AI热点速览 — 设计规格说明书

> 版本: v1.0 | 日期: 2026-06-14 | 状态: 待审批

---

## 1. 项目概述

### 1.1 一句话描述
面向中国 AI 开发工程师的每日 AI 新闻智能聚合 HTML 页面，基于 Claude Code 本地驱动，每日三次自动生成。

### 1.2 核心价值
- **信息过载解决方案**：从 6+ 信源自动聚合，去重、评分、翻译、排版一站式完成
- **中国开发者优先**：国产模型动态 + 政策合规重点覆盖，海外内容精选对中国有实际影响的
- **30 秒可扫读**：TL;DR 条 → 标题扫读 → 摘要精读 → 原文链接，四级信息深度

### 1.3 目标用户画像
- 中国 AI/大模型开发工程师（在职 + 转型中）
- 关注：模型选型、API 价格、国产硬件适配、开源工具链、政策合规
- 使用场景：早间通勤/咖啡时间（08:00版）、午休刷一眼（12:30版）、晚间复盘（20:00版）

## 2. 技术架构

### 2.1 运行模式
```
CronCreate 定时调度 (08:00 / 12:30 / 20:00)
       │
       ▼
Claude Code Session 启动
       │
       ├─→ WebSearch（多关键词并行搜索）
       ├─→ WebFetch（深度抓取原文）
       ├─→ GitHub/Gitee API（开源热度数据）
       │
       ▼
Claude 智能处理 Pipeline
  ① 去重（标题相似度 + URL 归一化）
  ② 评分（热度×0.4 + 时效×0.3 + 来源质量×0.3）
  ③ 翻译（海外新闻 → 中文摘要）
  ④ 分类（模型/工具/政策/论文/案例/开源/海外）
  ⑤ 生成摘要（每条 2-3 句，含开发者影响标注）
       │
       ▼
HTML 生成 & 归档
  • 生成当日版次 HTML
  • 更新 index.html（最新一期直显）
  • 更新 archive/ 历史文件
  • 更新日历导航数据
```

### 2.2 混合运行策略
- **Cron 自动模式**：承担日常 3 次采集，无人值守
- **手动 Skill 模式**：`/ai-digest` 命令，用于补采或重生成

### 2.3 信源矩阵
| 层级 | 信源 | 覆盖内容 |
|------|------|---------|
| 中文核心 | 36氪AI、量子位、机器之心、虎嗅 | 国内模型动态、行业政策、融资 |
| 中文技术 | 少数派、极客公园、即刻AI圈 | 工具框架、开发者经验 |
| 全球精选 | Hacker News、The Rundown AI | 海外重大动态（只选取影响中国的） |
| 开源 | GitHub Trending、Gitee 热门 | Stars 上升 + 关注度排行 |
| 学术 | arXiv cs.AI / cs.CL（最近48h） | 有落地价值的论文精选 |

### 3.2 视觉设计系统

**色彩**
- 背景：深邃渐变 `#0a0a0f → #0f0f1a → #0d0d18 → #0a0a14`
- 主色调：Indigo `#6366f1`（模型/品牌）
- 板块色：
  - 模型动态 `#6366f1`（Indigo）
  - 工具部署 `#22c55e`（Green）
  - 政策合规 `#f59e0b`（Amber）
  - 论文速递 `#8b5cf6`（Purple）
  - 开源热度 `#f59e0b`（Amber/Gold）
  - 海外参考 `#06b6d4`（Cyan）
- 文字：`#f5f5f7`（主） / `#9898a0`（次） / `#86868b`（辅助）
- 卡片：`rgba(255,255,255,0.025)` + `backdrop-filter: blur(16px)`

**字体**
- 西文：Inter（300/400/500/600/700）
- 中文：Noto Sans SC（300/400/500/600/700）
- 回退：-apple-system, BlinkMacSystemFont, sans-serif
- Hero 标题：44px / weight 280 / letter-spacing -0.03em
- 卡片标题：20px / weight 500

**动态背景**
- 4 个模糊渐变光球（Indigo/Purple/Cyan/Amber），18-24s 缓慢漂移
- 40 个随机微粒缓缓上升
- 60px 网格线 30s 缓慢偏移

**主题切换**
- 默认暗色模式
- 右上角经典滑块 Toggle（🌙 ○—— ☀️）
- 切换记忆（localStorage）
- 后续版本支持跟随系统 `prefers-color-scheme`

### 3.3 卡片交互
- Hover 时卡片上浮 2px + 边框亮度提升 + 阴影出现
- 过渡动画 0.3s ease
- 原文链接 hover 变色

## 4. 内容规格

### 4.1 板块内容量（按版次）

| 板块 | 早间(08:00) | 午间(12:30) | 晚间(20:00) | 格式 |
|------|:----------:|:----------:|:----------:|------|
| 🤖 大模型动态 | 1 Lead + 2-4 Sub | 增量1-2条 | 全天汇总5条 | 标题 + 2句摘要 + 来源 + 时间 + 💡开发者影响 |
| 🛠️ 工具&部署 | 3-4条 | 增量1-2条 | 汇总 | 工具名 + 一句话用途 + 新发布/更新标签 |
| 📋 政策&合规 | 1-2条 | 1条 | 汇总2条 | 政策名 + 影响简述 + 合规标签 |
| 📄 论文速递 | 1篇 | — | — | 标题 + 摘要 + arXiv链接 + 落地价值标注 |
| 💡 应用落地 | 2条 | 1条 | 2条 | 案例名 + 做法 + 效果 |
| ⭐ 开源热度 | 快照 | 快照 | Top10 | Stars增量 + 总关注度 |
| 🌍 海外参考 | 3条 | — | 3条 | 翻译摘要 + 来源链接 |

### 4.2 早午晚差异化
- **早间**：覆盖昨夜海外 + 国内早间，含论文速递（唯一版次），信息量最大
- **午间**：增量更新，只更新上午新出现的消息，减少 token 消耗
- **晚间**：全天汇总，开源 Top10 全天数据，海外参考，适合复盘

### 4.3 新闻分类标签
| 标签 | 颜色 | 用于 |
|------|------|------|
| `HEADLINE` | Indigo | 头条 Lead Story |
| `新发布` | Green | 新工具/模型发布 |
| `更新` | Amber | 版本更新/功能迭代 |
| `💰 融资` | Amber | 融资事件 |
| `📋 监管` | Blue | 政策法规 |
| `FOUNDATION MODELS` | 灰色 | 基础模型 |
| `OPEN SOURCE` | 灰色 | 开源相关 |
| `TOOLS` | 灰色 | 工具框架 |

- 默认展示最新一期速览

### 5.2 历史归档
```
archive/
├── 2026-06-13-morning.html
├── 2026-06-13-noon.html
├── 2026-06-13-evening.html

```
- 文件命名：`YYYY-MM-DD-{edition}.html`

- 列表视图：时间倒序，每期摘要预览

## 6. 错误处理

| 场景 | 处理策略 |
|------|---------|
| 某信源搜索失败 | 静默跳过，不阻塞其他源 |
| 全部信源失败 | 生成「暂无可获取新闻」页面，附最后成功时间 |
| 开源 API 限流 | 使用缓存的上次数据，标注数据时间 |
| HTML 生成失败 | 保留上一版，写错误日志 |
| Cron 错过触发 | 下次触发时补生成（最多补 1 期） |
